# Day 38 환경설정 - Barbershop Landing

## 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 목적 | Stitch + Claude Code 결합 워크플로우 테스트 |
| 앱 주제 | 바버샵 웹사이트 (영문) |
| 프로젝트명 | day38_barbershop_landing |
| 난이도 | Level 2 (프로세스 검증 목적) |
| 기술 스택 | Next.js + TypeScript + Tailwind CSS |
| 테스트 대상 | Stitch 목업(구조) + Dribbble(스타일) 분리 분석 |

---

## 1. 프로젝트 초기화

```bash
mkdir day38_barbershop_landing
cd day38_barbershop_landing
npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"
```

---

## 2. frontend-design 스킬 설치

```bash
# .claude/settings.json 에 추가 (Day 36에서 이미 설치했으면 생략)
{
  "permissions": {
    "allow": ["Read(mnt/skills/public/frontend-design/*)"]
  }
}
```

---

## 3. 레퍼런스 이미지 수집

```bash
mkdir -p docs/design/references/structure
mkdir -p docs/design/references/style
```

### Structure (Stitch 목업) ✅ 완료
11개 화면 → `docs/design/references/structure/`에 저장

### Style (Dribbble)
검색 키워드:
- "barbershop website"
- "dark luxury website"
- "gold accent web design"

2~4장 수집 → `docs/design/references/style/`에 저장

---

## 4. 데이터 준비

```bash
mkdir -p data
# services.csv, barbers.csv를 data/ 폴더에 복사
```

## 5. 폴더 구조

```

└── ...
```

---

## 6. 워크플로우

```
환경설정 ✅ (지금)
    ↓
SRS (Stitch 대화 기반)
    ↓
Design Analysis → structure/ + style/ 분리 분석
    ↓
CLAUDE.md → 60줄 이내
    ↓
IMPLEMENTATION.md → 페이즈별 구현 계획
    ↓
구현 → 페이즈별 npm run build 검증
```
